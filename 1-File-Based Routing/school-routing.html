Creating the Home Page (/app/page.js)

1. Project Setup

First, create a Next.js project:

npx create-next-app@latest school
cd school

2. Code for Home Page

Create page.js inside the app directory:

// app/page.js
export default function HomePage() {
  return (
    <div>
      <h1>Main Route: /app/page.js</h1>
    </div>
  );
}

3. Key Points

  • This creates a minimal home page with no styling or props
  • The component is automatically used as the root route ('/')
  • Access it at http://localhost:3000 after running npm run dev

Understanding Static Routes

What is a Static Route?

Static routes are the simplest form of routing in Next.js. They represent pages that don't change based on dynamic data or parameters. Each file in your app directory automatically becomes a route.

1. Creating Static Routes

Create files in your app directory following this structure:

school/

    ‚îú‚îÄ‚îÄ prep1/
    ‚îÇ   ‚îî‚îÄ‚îÄ page.js    // Prep1 route: /prep1
    ‚îú‚îÄ‚îÄ prep2/
    ‚îÇ   ‚îî‚îÄ‚îÄ page.js    // Prep2 route: /prep2
    ‚îî‚îÄ‚îÄ prep3/
        ‚îî‚îÄ‚îÄ page.js    // Prep3 route: /prep3

2. Example Static Route

Here's how to create a static Prep1 page:

// app/prep1/page.js
export default function Prep1Page() {
  return (
    <div>
      <h1>Prep1 Class</h1>
      <p>This is a static route at /prep1</p>
    </div>
  );
}

3. Key Points

  • File name must be page.js inside the route folder
  • The URL matches the folder structure
  • No special configuration needed - just create the file
  • Perfect for content that doesn't change often

4. Best Practices

  • Keep route names lowercase and use hyphens for multiple words
  • Group related routes in folders
  • Export the component as default
  • Use semantic HTML elements for better accessibility

Understanding Dynamic Routes

What is a Dynamic Route?

Dynamic routes in Next.js allow you to create pages that can handle dynamic data based on URL parameters. They're perfect for pages like user profiles, product details, or any content that follows a consistent pattern but with different data.

1. Creating Dynamic Routes

Create files using square brackets [param] to define dynamic segments:

school/
‚îî‚îÄ‚îÄ app/
    ‚îú‚îÄ‚îÄ students/
    ‚îÇ   ‚îî‚îÄ‚îÄ [id]/        // Dynamic segment
    ‚îÇ       ‚îî‚îÄ‚îÄ page.js  // Handles /students/1, /students/2, etc.
    ‚îî‚îÄ‚îÄ classes/
        ‚îî‚îÄ‚îÄ [grade]/     // Dynamic segment
            ‚îî‚îÄ‚îÄ page.js  // Handles /classes/prep1, /classes/prep2, etc.

2. Example Dynamic Route

Here's how to create a dynamic student profile page:

// app/students/[id]/page.js
export default function StudentPage({ params }) {
  return (
    <div>
      <h1>Student Profile</h1>
      <p>Student ID: {params.id}</p>
    </div>
  );
}

3. Key Points

  • Use square brackets [param] for dynamic segments
  • Access parameters via the params prop
  • Can have multiple dynamic segments: [grade]/[subject]/page.js
  • Works with any valid URL parameter value

4. Best Practices

  • Use meaningful parameter names
  • Validate parameters before using them
  • Handle loading and error states
  • Consider using TypeScript for better type safety

Understanding API Routes

What is an API Route?

API Routes in Next.js allow you to create API endpoints as part of your application. They provide a solution for building your API with Next.js, enabling you to handle HTTP requests and implement serverless functions.

1. Creating API Routes

Create route.js files in your app/api directory:

school/
‚îî‚îÄ‚îÄ app/
    ‚îî‚îÄ‚îÄ api/
        ‚îú‚îÄ‚îÄ grades/
        ‚îÇ   ‚îî‚îÄ‚îÄ route.js    // Handles /api/grades
        ‚îú‚îÄ‚îÄ attendance/
        ‚îÇ   ‚îî‚îÄ‚îÄ route.js    // Handles /api/attendance
        ‚îî‚îÄ‚îÄ schedule/
            ‚îî‚îÄ‚îÄ route.js    // Handles /api/schedule

2. Example API Route

Here's how to create a grades API endpoint:

// app/api/grades/route.js
export async function GET(request) {
  try {
    const grades = [
      { id: 1, student: "John", grade: "A" },
      { id: 2, student: "Sarah", grade: "B+" }
    ];
    
    return new Response(JSON.stringify(grades), {
      status: 200,
      headers: { 'Content-Type': 'application/json' }
    });
  } catch (error) {
    return new Response(JSON.stringify({ error: 'Failed to fetch grades' }), {
      status: 500,
      headers: { 'Content-Type': 'application/json' }
    });
  }
}

3. HTTP Methods

API routes support all HTTP methods:

// app/api/grades/route.js
export async function GET(request) { /* Handle GET */ }
export async function POST(request) { /* Handle POST */ }
export async function PUT(request) { /* Handle PUT */ }
export async function DELETE(request) { /* Handle DELETE */ }

4. Key Points

  • API routes are serverless functions
  • They run on the server side only
  • Support all HTTP methods (GET, POST, PUT, DELETE, etc.)
  • Can access databases and external services securely

5. Best Practices

  • Always validate input data
  • Use appropriate HTTP status codes
  • Implement proper error handling
  • Add authentication when needed
  • Use environment variables for sensitive data

Understanding Parallel Routes

What are Parallel Routes?

Parallel routes in Next.js allow you to simultaneously render multiple pages in the same layout. They're perfect for split views, dashboards, and modals.

1. Creating Parallel Routes

Use the @ prefix to define parallel routes:

app/

‚îÇ   ‚îî‚îÄ‚îÄ page.js     // URL: /
‚îî‚îÄ‚îÄ layout.js       // Renders both pages

2. Example Layout

// app/layout.js
export default function Layout({ 
  dashboard,  // @dashboard/page.js
  analytics   // @analytics/page.js
}) {
  return (
    <div className="grid">
      <div>{dashboard}</div>
      <div>{analytics}</div>
    </div>
  )
}

3. Key Points

  • Use @ prefix for parallel route folders
  • Each parallel route is independent
  • Can have their own loading and error states
  • Perfect for complex UI layouts

Understanding Route Interception

What is Route Interception?

Route interception allows you to intercept routes and show different content while maintaining the URL. Perfect for modals, slideovers, or custom views.

1. Creating Intercepted Routes

Use (..) convention to intercept routes:

app/
‚îú‚îÄ‚îÄ feed/
‚îÇ   ‚îî‚îÄ‚îÄ page.js           // URL: /feed
‚îî‚îÄ‚îÄ photo/
    ‚îú‚îÄ‚îÄ [...id]/
    ‚îÇ   ‚îî‚îÄ‚îÄ page.js       // URL: /photo/123
    ‚îî‚îÄ‚îÄ (..)feed/
        ‚îî‚îÄ‚îÄ [id]/
            ‚îî‚îÄ‚îÄ page.js   // Intercepts /photo/123 
                         // while on /feed

2. Example Intercepted Route

// app/photo/(..)feed/[id]/page.js
export default function PhotoModal({ params }) {
  return (
    <div className="modal">
      <img src={`/photos/${params.id}`} />
    </div>
  )
}

3. Key Points

  • Use (..) to go up one level
  • Use (...) to match segments from the root
  • Perfect for modal patterns
  • Maintains clean URLs
